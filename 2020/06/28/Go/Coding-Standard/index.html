Go语言编码规范

本规范旨在为日常Go项目开发提供一个代码的规范指导，方便团队形成一个统一的代码风格，提高代码的可读性、规范性和统一性。

一、命名规范

命名是代码规范中很重要的一部分，统一的命名规则有利于提高的代码的可读性，好的命名仅仅通过命名就可以获取到足够多的信息。

Go在命名时以大小写字母a到z或下划线开头，后面可跟着字母、下划线和数字(0~9)。

Go不允许在命名时使用@、$和%等标点符号。

Go是一种区分大小写的编程语言。

  1. 当命名（包括变量、常量、类型、函数、结构字段等等）以一个大写字母开头，如: Group1，那么使用以这种形式的标识符的对象就可以被外部包的代码所使用（客户端程序需要先导入这个包），这被称为导出（类似面向对象编程语言中的 public）；
  2. 命名如果以小写字母开头，则对外部包是不可见的，但是他们在整个包的内部是可见并且可用的（类似面向对象编程语言中的 private）

1、包命名：package

保持 package 的名字与目录保持一致，尽量采取有意义的包名，简短，有意义，尽量和标准库不要冲突。包名应该为小写单词，不要使用下划线或者混合大小写。

2、文件命名

尽量采取有意义的文件名，简短，有意义，应该为小写单词，使用下划线分割各个单词。

3、结构体命名

  • 采用驼峰命名法，首字母根据访问控制大写或小写
  • struct 申明和初始化格式采用多行

4、接口命名

  • 命名规则基本与接口体类型类似
  • 单个函数的结构名以er作为后缀，例如 Reader, Writer

5、变量命名

  • 和结构体类似，变量名称一般遵循驼峰法，首字母根据访问控制原则大写或者小写，但遇到特有名词时，需要遵循一下规则：
    • 如果变量为私有，且特有名词为首个单词，则使用小写，如 apiClient
    • 其他情况都应当使用该单词原有的写法，如 APIClient、repoID、UserID
    • 错误示例：UrlArray，应该写成 urlArray 或 URLArray
  • 若变量类型为 bool 类型，则名称以 Has, ls, Can 或 Allow 开头

6、常量命名

常量均需使用全部大写字母组成，多个单词使用下划线分隔。

如果是枚举类型的常量，需要先创建相应的类型

const fallthrough if range type

二、代码风格

1、语句的结尾

Go 语言中语句结尾不需要分号结尾，默认一行就是一条数据。

如果打算将多个语句写一行，则必须使用 ;

2、括号和结尾

所有的运算符和操作数之间要留空格

4、错误处理

  • 错误处理的规则就是不能丢弃任何有返回 err 的调用，不要使用 _ 丢弃，必须全部处理。接收到错误，要么返回 err，要么使用log记录下来。
  • 尽早 return 一旦有错误发生，马上返回
  • 尽量不要使用 public，除非你知道你在做什么
  • 错误描述如果是英文必须为小写，不需要标点结尾
  • 采用独立的错误流进行处理

5、测试

单元测试文件命名规范为 example_test.go

测试用例的函数名必须以 Test 开头, 例如：TestExample

每个重要的函数都要首先编写测试用例，测试用例和正规代码一起提交方便进行回归测试

三、常用工具

  • gofmt - 自动格式化代码，保证所有的 go 代码与官方推荐的格式保持一致。
  • goimport - 该工具在 gofmt 的基础上增加了自动删除和引入包。
  • go vet - 该工具可以帮我们静态分析我的的源码存在的各种问题，例如多余的代码，提前 return 的逻辑，struct 和 tag 是否符合规范等。
